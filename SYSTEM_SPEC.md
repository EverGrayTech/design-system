# EverGray Tech
## Visual System Specification

### Purpose
This specification defines the shared visual system for `EverGray Tech`, both it's website [evergraytech.com](https://evergraytech.com) and all the company's applications. It establishes a consistent design language that can be implemented across products without additional interpretation.

## 1. Design Intent

The visual system should communicate:

- technical competence
- clarity under complexity
- premium execution
- geometric precision
- forward-looking capability
- disciplined restraint

The system must feel advanced without feeling trendy. It must feel refined without feeling ornamental. It must feel premium because of precision, not because of extravagance.

### Core phrase
**Engineered Elegance**

### Design family
**Geometric Minimalism + Premium Technical**

### Behavioral model
The company website is the slightly more cinematic expression of the system.
The applications are the stricter expression of the system.

## 2. Brand Character

### Desired qualities
- precise
- technical
- trustworthy
- calm
- advanced
- efficient
- sophisticated
- deliberate

### Undesired qualities
- gimmicky
- playful
- flashy
- soft
- blob-like
- over-animated
- cyberpunk
- decorative for its own sake
- luxury-brand posturing

### Product posture
The products are customer-facing workflow and productivity applications, primarily serving developers and technical users. Interfaces should support concentration, speed, and confidence.

## 3. Core Visual Principles

### 3.1 Clarity over flourish
Every visual choice should improve comprehension, orientation, or confidence. Avoid visual treatments that exist only to create atmosphere.

### 3.2 Restraint creates value
Brand expression should be selective. The more disciplined the baseline system is, the more effective accent and motion become.

### 3.3 Geometry is identity
The logo’s geometric character should inform the overall system: circles, straight lines, exact alignment, consistent radii, and mathematically ordered composition.

### 3.4 Motion is feedback
Motion should reinforce causality, hierarchy, and state change. Motion is not decoration.

### 3.5 Layering should feel machined
Surfaces should feel separated through tonal control and subtle edge response rather than obvious outlining or heavy shadow.

### 3.6 Premium means composed
The interface should feel expensive because it is controlled, balanced, and intentional. Never because it is loud.

## 4. Overall Style Direction

### Foundation
Use a charcoal-based dark theme with subtle tonal separation between structural layers.

### Character
The visual language should feel:
- mathematically aligned
- precision-machined
- calm under load
- information-efficient without becoming dense or aggressive

### Expression hierarchy
1. structure
2. typography
3. tonal layering
4. accent
5. motion

Accent and motion must never carry the interface by themselves.

## 5. Color System

### 5.1 Base palette
The foundation of the system is a neutral dark palette centered on charcoal.

Use multiple dark surface levels rather than a single background tone. The system should clearly distinguish:
- page canvas
- application chrome
- standard surfaces
- elevated surfaces
- overlays

These distinctions should be subtle but immediately legible.

### 5.2 Surface contrast
Surface contrast should be controlled and measured. Avoid harsh separation between adjacent layers. The system should feel layered, not striped.

### 5.3 Accent posture
Accent should feel advanced, not playful.

The company’s red / dark blue / light blue gradient is the signature accent language. It should be treated as special and purposeful, not routine.

Use accent primarily for:
- brand moments
- primary calls to action
- hero emphasis
- selected or focused moments where brand emphasis is appropriate
- progress or transformation states
- diagrams and controlled graphic highlights

Do not use accent as the default treatment for ordinary interface structure.

### 5.4 Semantic colors
Semantic feedback states must remain clear and independent from brand accent. Success, warning, error, and informational states should function reliably without borrowing the brand gradient language.

### 5.5 Color behavior
The interface should be mostly neutral. Color should punctuate the experience, not saturate it.

## 6. Surface Model

### 6.1 Surface language
Use subtle background differences to create a layered spatial model.

Surfaces should feel like refined planes within a shared system, not disconnected cards floating on a dark background.

### 6.2 Edge treatment
Where a stroke is needed, it should read more like a faint glint on the edge than an explicit outline.

Borders should be understated, cool-neutral, and low-contrast.

### 6.3 Depth model
Depth should be established primarily through:
- tonal separation
- edge light
- containment
- spacing
- occasional restrained shadow where structurally necessary

Depth should not rely on:
- thick outlines
- soft floating-card shadows
- glow effects
- glassmorphism

### 6.4 Structural lines
Dividers and line work may be more deliberate than surface edges, but only when they serve structure or information. This allows line-based graphics, diagrams, and layout separators to feel purposeful rather than redundant.

## 7. Typography

### 7.1 Typographic direction
Typography should feel premium-tech: modern, highly legible, precise, and controlled.

It should lean toward a contemporary technical sans with disciplined proportions and strong screen readability. The overall tone should be closer to refined developer tooling than editorial luxury or consumer friendliness.

### 7.2 Typographic behavior
Typography carries authority in this system. It should do most of the work of hierarchy.

Headlines should feel confident, clean, and composed.
Body text should be calm and highly readable.
UI text should be efficient, crisp, and unobtrusive.
Monospaced type may be used selectively for technical affordances or data-like elements, but must not dominate the visual language.

### 7.3 Hierarchy
Hierarchy should come from:
- scale
- weight
- spacing
- placement
- contrast

Hierarchy should not depend on decorative styling.

### 7.4 Density
The applications should support information efficiency without feeling crowded. Typography should allow compactness where needed, but the system should never feel compressed or frantic.

## 8. Geometry and Shape

### 8.1 Geometric alignment
Layouts, components, and visual motifs should feel mathematically aligned. Composition should reflect precision rather than spontaneity.

### 8.2 Shape language
The system should use restrained geometric forms:
- straight lines
- circles or circular references where intentional
- modest corner radii
- repeated proportional relationships

### 8.3 Radii posture
Shapes should feel precision-machined, not soft or toy-like.

Use restrained rounding. Avoid over-rounded consumer SaaS softness and avoid organic or irregular forms.

### 8.4 Graphic motifs
Illustration, diagrams, and support graphics should follow the same geometric discipline as the interface. They should use line, structure, and controlled accent rather than decorative abstraction.

## 9. Layout and Spacing

### 9.1 Layout character
Layouts should feel ordered, measured, and intentional. Use grid discipline throughout both marketing and product surfaces.

### 9.2 Spacing behavior
Spacing should create calm and readability. It should clarify grouping, reduce visual noise, and support scanning.

### 9.3 Balance
The system should avoid both extremes:
- not cramped or dense by default
- not oversized or airy to the point of feeling like marketing theater

### 9.4 Composition
The website may use more dramatic composition than the applications, but both should remain controlled and geometric.

## 10. Motion

### 10.1 Motion philosophy
Motion should feel subtle and physically plausible. It should read as system feedback, not performance.

### 10.2 Appropriate uses of motion
Use motion to communicate:
- interaction feedback
- entrance and exit
- focus change
- selection
- hierarchy
- continuity between states

### 10.3 Inappropriate uses of motion
Do not use motion for spectacle, novelty, or ambient distraction.

Avoid:
- heavy hero choreography
- gratuitous parallax
- bouncing or playful motion
- continuous environmental movement in product surfaces
- effects that compete with information

### 10.4 Website motion
The website may use subtle animated gradient fields or restrained atmospheric motion in hero sections. This motion must remain low-amplitude, smooth, and secondary to content.

### 10.5 Application motion
Applications should use a quieter motion profile focused on responsiveness and state clarity.

## 11. Website vs. Application Expression

### 11.1 Shared system
The website and applications must clearly belong to the same company. They should share:
- the same foundational dark palette
- the same accent logic
- the same typographic family and hierarchy principles
- the same shape language
- the same spacing philosophy
- the same motion philosophy

### 11.2 Website expression
The website is the slightly more cinematic expression of the system.

It may include:
- more spacious composition
- larger typographic moments
- controlled atmospheric gradient usage
- stronger brand storytelling
- a small increase in visual drama

### 11.3 Application expression
The applications are the stricter expression of the system.

They must prioritize:
- clarity
- information efficiency
- focus
- structural calm
- restrained accent usage
- minimal decorative motion

Applications should feel operational first and branded second.

## 12. Component-Level Behavioral Guidance

### 12.1 Controls
Controls should feel precise, reliable, and crisp. They should not look inflated, glossy, or playful.

### 12.2 Navigation
Navigation should be quiet but confident. Selection and focus should be unmistakable without becoming loud.

### 12.3 Panels and cards
Panels should be defined through subtle layer contrast and restrained edge treatment. They should feel embedded in a coherent system, not independently styled.

### 12.4 Inputs
Inputs should prioritize legibility and focus clarity. Interaction states should be obvious, but the shell should remain understated.

### 12.5 Data-dense surfaces
Lists, tables, and workflow-heavy views should use hierarchy, spacing, and controlled separators to preserve readability. Avoid ornamental chrome.

### 12.6 Empty states and supporting graphics
Use geometric, line-led, restrained visuals. Avoid mascots, organic shapes, or playful illustration styles.

## 13. Brand Graphics

### 13.1 Graphic tone
Brand graphics should feel technical, geometric, and deliberate.

### 13.2 Line work
Purposeful line work is encouraged in diagrams, dividers, frameworks, and abstract support graphics, provided it remains restrained and aligned with the surface model.

### 13.3 Gradient use in graphics
The gradient may play a stronger role in graphics than in UI chrome, but it must still remain structured and controlled. Use it to emphasize direction, transformation, or focal energy.

### 13.4 Imagery
When imagery is used, it should reinforce competence and product substance. Favor product, system, and concept visuals over lifestyle imagery.

## 14. Accessibility and Usability Expectations

The dark theme must remain highly usable in extended sessions.

Ensure:
- readable contrast at all text levels
- clear distinction between layers and states
- obvious focus indication
- semantic states that remain legible and distinct
- low cognitive overhead in dense screens

The system must not sacrifice usability for atmosphere.

## 15. Non-Negotiable Rules

- Do not use blob shapes.
- Do not use heavy animation.
- Do not use the brand gradient as a default fill language.
- Do not rely on glow as a structural device.
- Do not use thick visible outlines as the primary edge treatment.
- Do not make interfaces feel soft, playful, or toy-like.
- Do not allow marketing styling to leak into application core surfaces.
- Do not make the dark theme harsh through excessive black-and-white severity.

## 18. Summary Directive

Build a charcoal-based dark visual system that feels geometric, advanced, and premium through precision rather than spectacle. Use typography, spacing, tonal layering, and exact alignment as the primary tools of sophistication. Treat the brand gradient as a special signal, not a constant presence. Let the website carry slightly more atmosphere, while the applications remain calm, efficient, and operational.

The result should feel like a refined technical instrument: deliberate, capable, and unmistakably modern.
